---
description: photos/ 폴더의 새 관 사진을 읽어 data.js에 좌석 데이터 추가 후 GitHub 푸시
---

1. `photos/*/` 폴더 목록과 `data.js`의 기존 극장·관을 비교해 아직 없는 폴더를 찾는다.
   폴더명 규칙: `지점명_관이름` 또는 `지점명_관이름_태그` (태그는 관이름 뒤에 "관이름 (태그)"로 표기).
2. 새 폴더의 이미지를 Read로 전부 읽고 좌석 배치를 파악한다:
   - rows: 열 라벨(A, B, ...)과 열별 좌석 수
   - aisles: 통로 위치 (n번째 좌석 뒤)
   - screenWidth / firstRowDist / rowSpacing: 사진에서 추정. 확신 없으면 관 규모 기준 상식값
     (일반관: 10~12m / 6m / 1.1m, IMAX·대형관: 20m+ / 10m / 1.2m, 소형관: 8m / 4m / 1.0m)
     을 쓰고 해당 줄에 `// 추정치` 주석을 단다.
3. data.js에 추가한다. 같은 지점이 이미 있으면 그 지점의 screens 배열에 관만 추가.
4. 파악한 배치(열 수, 좌석 수, 통로)를 사용자에게 요약해 보여준다. 사진이 흐릿하거나
   배치를 확신할 수 없으면 추측으로 넣지 말고 사용자에게 물어본다.
5. 커밋하고 push한다. 커밋 메시지: `data: <추가된 관 목록>`

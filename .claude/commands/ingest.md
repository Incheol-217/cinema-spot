---
description: photos/ 의 새 관 사진을 읽어 data.js에 좌석 데이터 추가 후 GitHub 푸시
---

1. `photos/` 안의 이미지 파일과 폴더를 나열하고, `data.js`의 기존 극장·관과 비교해
   아직 없는 것을 찾는다. 이름 규칙: `영화관종류_지점_관`
   (예: `CGV_의정부_IMAX관` → 극장 "CGV 의정부", 관 "IMAX관"). 파일 하나든 폴더든 허용.
2. 이미지를 Read로 전부 읽고 좌석 배치를 파악한다:
   - rows: 열 라벨(A, B, ...)과 열별 좌석 수. 없는 자리는 `absent` 배열로.
   - 판매완료·선택됨 등으로 가려진 자리는 주변 열·번호를 보고 일반석으로 채운다.
   - aisles: 통로 위치 (n번째 좌석 뒤)
   - centerCol: 스크린 중심이 놓인 좌석 번호 (열마다 길이가 다르면 필수)
   - screenWidth / firstRowDist / rowSpacing: 사진에서 추정. 확신 없으면 관 규모 기준 상식값
     (일반관: 10~12m / 6m / 1.1m, IMAX·대형관: 20m / 9m / 1.2m, 소형관: 8m / 4m / 1.0m)
     을 쓰고 해당 줄에 `// 추정치` 주석을 단다.
3. data.js에 추가한다. 같은 지점이 이미 있으면 그 지점의 screens 배열에 관만 추가.
4. 파악한 배치(열 수, 좌석 수, 통로, 채워 넣은 자리)를 사용자에게 요약해 보여준다.
   사진이 흐릿하거나 배치를 확신할 수 없으면 추측으로 넣지 말고 사용자에게 물어본다.
5. 커밋하고 push한다. 커밋 메시지: `data: <추가된 관 목록>`
